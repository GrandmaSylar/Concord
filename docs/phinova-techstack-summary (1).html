Tech Stack Summary
Bulk SMS Platform · May 2026

This document outlines the 7 technologies that power the PhiNova Bulk SMS Platform — what each one is, what role it plays in the project, and what it costs. Every tool was chosen to be scalable, fast, and free to run — with Arkesel credits being the only real ongoing expense.

⚛️ Next.js + Tailwind CSS
React Framework
The Face of the Platform
Every screen the user sees — the dashboard, contact list, template editor, send page, and reports — is built here. Next.js handles routing, page rendering, and performance optimisation automatically. Tailwind CSS styles everything without writing a single custom CSS file. Unlike plain HTML, only the parts of the page that change re-render — making the UI feel instant and app-like.
  • Faster than HTML — no full page reloads on navigation
  • Scales cleanly as new pages and features are added
  • Deployed and optimised automatically by Vercel
🟢 Node.js + Express.js
Backend Server
The Brain of the Platform
Handles all the logic running behind the scenes. When a user hits Send SMS, Express receives that request, validates it, calls the Arkesel API, and returns the delivery result. Every action on the frontend — sending messages, saving contacts, managing templates — flows through an Express route.
  • Processes all API calls between the frontend and external services
  • Talks directly to Supabase to read and write data
  • Runs as serverless functions on Vercel — no server to manage
🗄️ Supabase — PostgreSQL Database
Cloud Database
The Memory of the Platform
Stores everything permanently in the cloud — contacts, message templates, sent message history, scheduled reminders, and user accounts. Because it's cloud-hosted, data is never lost between deploys or server restarts. The free tier provides 500MB of storage — more than enough for thousands of contacts and message records.
  • Replaces SQLite — data persists reliably in the cloud
  • Also handles CSV file storage for contact imports (1GB free)
  • Secure, backed up, and accessible from anywhere
🔐 Supabase Auth
Authentication
The Gatekeeper
Manages who can log in and what they can access. Handles password hashing, session management, and role-based access (admin vs standard user) — all out of the box. Replaces the need for bcrypt and express-session entirely, with far more security built in.
  • Admins can manage contacts, templates, and users
  • Standard users can send messages and view reports only
  • Row-Level Security ensures users only see their own data
⏰ Supabase Edge Functions
Scheduler
The Timekeeper
Runs scheduled jobs automatically inside Supabase — no separate server or cron service needed. Every day it checks the database for contacts with birthdays, anniversaries, or appointments due, then calls the Arkesel API to send the right message at the right time. Because it lives inside Supabase, it has direct, instant access to the database with zero latency.
  • Fires reminders automatically — set once, runs forever
  • Direct database access — no external API calls needed to read contacts
  • Runs on a schedule as tight as every 1 minute
📱 Arkesel API
SMS Gateway
The Messenger
The only service that actually sends SMS messages to phone numbers through the telecom network. Express sends Arkesel a recipient number and message text — Arkesel handles everything else, including delivery, sender ID management, and returning a delivery status. The platform is entirely invisible to the recipient — they just receive the message.
  • Handles bulk sending to full contact lists in one API call
  • Returns real-time delivery confirmations per message
  • Credit balance is displayed live inside the dashboard
🚀 Vercel
Hosting & Deployment
The Landlord
Hosts and serves the entire platform live on the internet. Built by the same team as Next.js, so the two work together seamlessly — automatic HTTPS, global CDN, and instant deploys every time code is pushed to GitHub. The free Hobby tier covers this project entirely with no credit card required to start.
  • Global CDN means fast load times regardless of where users are
  • Auto-deploys on every GitHub push — no manual uploads
  • Now viable because data lives in Supabase, not on local disk
